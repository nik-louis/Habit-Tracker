# 🥕 Carrot &amp; Stick — Реализация

Однофайловый трекер привычек с геймификацией, зональной системой штрафов/наград, многолетним учётом и зашифрованной облачной синхронизацией. Всё приложение — это один `index.html` без сборки, бэкенда и зависимостей.

> Личная система достижения целей по принципу «кнут и пряник»: закрываешь нормы — получаешь пряники, проваливаешь — деньги уходят в благотворительность.

---

## ✨ Возможности

- **19 привычек** с тегами по сферам жизни (биохакинг, влияние, отношения, Valorant, душа, финансы), весами и индивидуальными настройками.
- **Геймификация:** 100 уровней, 20 рангов (Novice → God), опыт за каждую закрытую привычку, заморозка серий.
- **Зональная система (в процентах, гибридная):**
  - 🥕 Награда (≥85%) → пряник
  - ⚪ Буфер (65–85%) → без штрафа
  - 🟠 Кнут ×1 (45–65%)
  - 🔴 Кнут ×2 (25–45%)
  - ⛔ Кнут ×2 + рост ставки на следующий месяц (&lt;25%)
  - Пороги настраиваются глобально и по каждой привычке, переключение «проценты ↔ дни».
- **Благотворительная модель штрафов:** фиксированный % дохода уходит в «штрафной банк», делится по весам привычек, потерянное — на благотворительность.
- **Магазин больших наград:** маленькие пряники = валюта, копишь и обмениваешь на крупные награды. Полное редактирование (добавление/изменение/удаление).
- **Многолетний учёт (2026–2050):** каждый год — чистый лист (галочки, XP, доход, аналитика свои), а привычки, настройки и награды общие. Корректные дни недели и високосные годы.
- **Аналитика:** дисциплина по месяцам, недельные паттерны, эффективность по тегам, серии, тепловая карта, корреляции «эффект следующего дня», достижения, итоги месяца.
- **Облачная синхронизация (Firebase) с шифрованием AES-256-GCM:** данные шифруются на устройстве, в облаке — нечитаемый шифротекст. Ключ выводится из пароля через PBKDF2.
- **Экспорт:** JSON, CSV, PDF, PNG — полностью локально.
- **Темы** (тёмная/светлая), акцентные цвета, заметки на день, архив/удаление с сохранением истории, версии и восстановление.

## 🚀 Запуск

### Локально
Просто открой `index.html` в браузере. Данные сохраняются в `localStorage`.

### Как сайт (GitHub Pages)
1. Settings → Pages → Source: `Deploy from a branch` → Branch: `main` / `root` → Save.
2. Через минуту приложение будет доступно по адресу `https://<твой-логин>.github.io/<репозиторий>/`.

### Через Netlify
Перетащи `index.html` на [app.netlify.com](https://app.netlify.com) — получишь ссылку за 15 секунд.

## ☁️ Облачная синхронизация (необязательно)

Нужна только если хочешь синхронизировать данные между устройствами.

1. Создай бесплатную базу [Firebase Realtime Database](https://console.firebase.google.com).
2. На вкладке **Rules** вставь и опубликуй:
   ```json
   {
     "rules": {
       "sync": {
         "$c": {
           ".read": "$c.length >= 16",
           ".write": "$c.length >= 16"
         }
       }
     }
   }
   ```
3. В приложении: ⚙️ Настройки → **Cloud Sync** → введи:
   - **Database URL** — адрес из Firebase
   - **Sync Code** — общий код (≥16 символов)
   - **🔒 Пароль шифрования** — твой личный пароль
4. На втором устройстве введи те же три значения (или используй кнопку «Copy key for 2nd device»).

> 🔐 **Безопасность:** данные шифруются на устройстве (AES-256-GCM, ключ через PBKDF2). В облако попадает только зашифрованный конверт `{enc, iv, ct}`. Без пароля данные не прочитать даже при полном доступе к базе. **Забытый пароль не восстанавливается.**

## 🔐 Как устроено шифрование

Код шифрования открыт (он в `index.html`), но это безопасно — по принципу Керкгоффса стойкость держится не на секретности алгоритма, а на пароле. Пароль **не хранится в файле**: он вводится пользователем и живёт только в его браузере (`localStorage`). У каждого пользователя свой пароль, своя база и свои данные — общий только код приложения.

| Слой | Что используется |
|---|---|
| Вывод ключа | PBKDF2, SHA-256, 120 000 итераций, соль из Sync Code |
| Шифрование | AES-256-GCM со случайным IV на каждую запись |
| Хранение в облаке | `{enc:1, v:1, iv, ct, _t}` — без открытого текста |
